## 🚀 超快速开始（3步）

### 第1步：安装依赖

打开终端，进入项目目录，运行：

```bash
./setup.sh
```

这个脚本会自动：
- ✅ 检查您的Node.js环境
- ✅ 安装所有后端依赖
- ✅ 安装所有前端依赖
- ✅ 创建环境配置文件

**预计耗时**: 2-3分钟

### 第2步：启动服务

```bash
./start.sh
```

这个脚本会自动：
- ✅ 清理可能占用的端口
- ✅ 同时启动前后端服务
- ✅ 显示访问地址

**预计耗时**: 5-10秒

### 第3步：访问系统

打开浏览器，访问：

**http://localhost:3000**

🎊 恭喜！您已成功启动系统！

## 🎯 系统能做什么？

这个系统帮助您回答一个核心问题：

> **"哪些资源需要扩容，什么时候必须扩，优先级是什么？"**

### 4个角色视角

我们为不同角色提供定制化的报表：

#### 1️⃣ CTO视角
查看技术扩容方案：
- 哪些组件需要扩容？
- 怎么扩？（具体方案）
- 需要多少工作量？

#### 2️⃣ CEO视角
了解业务影响：
- 会不会影响业务？
- 值不值得扩容？
- 什么时候必须扩？

#### 3️⃣ 运维总监视角
全局容量规划：
- 全局排序
- 时间安排
- 资源需求

#### 4️⃣ 运维工程师视角
详细执行指导：
- 具体怎么做？
- 验证方法
- 风险提示

## ❓ 遇到问题？

### 端口被占用

**症状**: 启动失败，提示端口3000或3001被占用

**解决**: 运行 `./start.sh`（会自动清理）

或手动清理：
```bash
lsof -ti:3001 | xargs kill -9
lsof -ti:3000 | xargs kill -9
```

### 模块找不到

**症状**: 启动时提示 "Cannot find module"

**解决**: 重新安装依赖
```bash
./setup.sh
```

### 浏览器显示空白

**症状**: 打开 http://localhost:3000 显示空白页

**解决**: 
1. 按 F12 打开浏览器控制台查看错误
2. 按 Ctrl+Shift+R 强制刷新
3. 检查终端是否有编译错误

### 更多问题？

查看详细的故障排除文档：
```bash
cat TROUBLESHOOTING.md
```
