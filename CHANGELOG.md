## [2.0.0] - 2024-11-21

### 🎉 重大改进：本地开发环境优化

这个版本专注于改善本地开发体验，让用户能在几分钟内完成安装和启动。

### ✨ 新增功能

#### 自动化脚本
- **setup.sh** - 一键安装所有依赖和配置
  - 自动检测Node.js环境
  - 安装后端和前端依赖
  - 自动创建环境配置文件
  - 友好的进度提示
  
- **start.sh** - 智能启动脚本
  - 自动检查依赖是否安装
  - 自动清理被占用的端口（3000/3001）
  - 同时启动前后端服务
  - 清晰的访问地址提示
  
- **quick-test.sh** - 快速API验证
  - 测试所有5个API端点
  - 显示HTTP状态码
  - 展示示例数据
  - 彩色输出（✅/❌）

#### 环境配置文件
- **.env.example** - 后端环境配置模板
- **.env** - 后端环境配置（自动创建）
- **client/.env.example** - 前端环境配置模板
- **client/.env** - 前端环境配置（自动创建）

所有配置文件都使用合理的开发环境默认值。

## 升级指南

### 从 v1.0 升级到 v2.0

#### 1. 更新代码
```bash
git pull origin main
```

#### 2. 创建环境文件
```bash
cp .env.example .env
cp client/.env.example client/.env
```

#### 3. 重新安装依赖（可选）
```bash
./setup.sh
```

#### 4. 使用新脚本启动
```bash
./start.sh
```

### 注意事项

- 所有原有功能保持兼容
- API端点无变化
- 数据库Schema无变化
- 可以继续使用旧的启动方式（npm run dev）
